# FitbitAir — مزامنة مع آخر نسخة من البوت

هذه الحزمة مبنية على آخر نسخة أرسلها أحمد:
`fitbit-bot-main افضل تعديل واخر تعديل.zip`

## ما تم عمله

- الحفاظ على جميع تعديلات البوت الأخيرة، بما فيها:
  - إعادة ربط Google Health
  - token_store.py
  - مسارات reauth
  - التنبيهات والـcron
  - واجهات Mini App الحالية
  - سجل التمارين وقاعدة البيانات الحالية
- إضافة API تطبيق iPhone فوق آخر نسخة، بدون الرجوع إلى ملفات السيرفر القديمة.
- إبقاء تطبيق iPhone على نفس رابط Railway الحالي.
- إبقاء التطبيق والبوت على نفس قاعدة البيانات ونفس AI ونفس مصدر Google Health.
- إبقاء Workflow بناء IPA الموثّق.

## الرفع

### مشروع Railway القديم
ارفع **محتويات** مجلد `Railway-Server` مباشرة فوق ملفات مشروع GitHub المرتبط بـRailway.

### مشروع تطبيق iPhone
ارفع:
- `FitbitAir-iOS`
- `.github`

إلى Repository بناء التطبيق، ثم شغّل GitHub Actions.

## متغير التطبيق الشخصي

التطبيق يستخدم نفس مفتاح API الحالي. السيرفر يدعم:
`IOS_API_KEY`

إذا لم تضفه في Railway، سيستخدم المفتاح الشخصي الحالي المضمّن كقيمة افتراضية للتوافق مع التطبيق الحالي.
