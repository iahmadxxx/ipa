# دليل نشر البوت على Railway 🚂

## الملفات بهذي الحزمة
- `webhook_app.py` — تطبيق Flask (أوامر البوت + الصوت + تمارين الحديد)
- `gym_tracker.py` — تتبّع تمارين الحديد (قاعدة بيانات SQLite دائمة)
- `google_health_client.py` — جلب البيانات من Google Health
- `analyzer.py` — تنسيق الرسائل العربية
- `ai_coach.py` — المدرب الذكي (Gemini) + تحويل الصوت لنص
- `daily_summary.py` — الرسالة الصباحية (خدمة مجدولة)
- `inactivity_check.py` — تذكير الحركة (خدمة مجدولة جديدة)
- `requirements.txt` + `Procfile` — ملفات Railway

### 🆕 جديد بهذي النسخة
- **تتبّع تمارين الحديد** 🏋️ — اكتب `/تمرين` → تختار يوم (صدر وباي / ظهر وتراي / أكتاف / أرجل) → تختار تمرين → ترسل كل جولة "تكرار وزن" مثل `15 20` → تكتب "خلص" لما تخلص. يحفظ كل شي عشان يرجع لك تاريخك في كل مرة (يعرض لك آخر مرة سويت فيها نفس التمرين تلقائيًا). **يحتاج خطوة Volume بالأسفل عشان البيانات ما تضيع.**
- **محادثة حرة مع المدرب الذكي** 💬 — ما تحتاج `/coach` أو `/analyze` بعد الحين؛ أي رسالة عادية ترسلها (مو أمر يبدأ بـ `/`) يفهمها البوت كسؤال ويجاوبك مباشرة بناءً على بياناتك، مثل: "حلل نومي"، "وش أفضل وقت للكارديو؟"، "ليش نبضي مرتفع؟".
- **أوامر صوتية** 🎙️ — أرسل رسالة صوتية بدل الكتابة، والبوت يحوّلها لنص ويرد عليك (ولو ما كانت أمر معروف، تترّس تلقائيًا كسؤال حر للمدرب).
- **تذكير تلقائي بالحركة** 🚶 — لو خطواتك أقل من المتوقع لوقت اليوم، يرسل لك تنبيه (يحتاج خدمة Cron جديدة، شوف الخطوة 6.1 تحت).

⚠️ لاحظ: تم إلغاء الأزرار الثابتة اللي كانت تحت مربع الكتابة (كانت تسبب مشكلة). الحين تستخدم الأوامر مباشرة، وتمارين الحديد لها أزرارها الخاصة اللي تطلع بعد `/تمرين`.

⚠️ لاحظ: **ما فيه env_config.py هنا** — Railway يستخدم Variables من لوحة التحكم مباشرة (أأمن وأنظف).

## الخطوة 1: ارفع الملفات على GitHub

Railway ينشر من GitHub، فنحتاج مستودع (repo):

1. سجّل/ادخل على [github.com](https://github.com).
2. اضغط **+** (فوق يمين) → **New repository**.
3. الاسم: `fitbit-bot` → اختر **Private** (مهم! عشان كودك ما يكون عام) → **Create repository**.
4. بصفحة المستودع الفاضي، اضغط رابط **uploading an existing file**.
5. اسحب/ارفع الملفات السبعة كلها → اكتب أي وصف → **Commit changes**.

## الخطوة 2: أنشئ المشروع على Railway

1. من [railway.com](https://railway.com) بعد تسجيلك بخطة Hobby.
2. **New Project** → **Deploy from GitHub repo**.
3. أول مرة بيطلب ربط حساب GitHub — اسمح له بالوصول لمستودع `fitbit-bot`.
4. اختر المستودع → Railway بيبدأ يبني وينشر تلقائيًا (يكتشف Python والـ Procfile بنفسه).

## الخطوة 2.5: خزّن بيانات تمارينك بشكل دائم (Volume) 💾

مهم جدًا! بدون هالخطوة، Railway يمسح ملف بيانات التمارين كل ما ننشر تحديث جديد للكود.

1. بنفس الخدمة (Service) اللي نشرت فيها البوت → تبويب **Settings**.
2. انزل لقسم **Volumes** → اضغط **+ New Volume**.
3. بخانة **Mount Path** اكتب: `/data` → احفظ.
4. Railway بيعيد نشر الخدمة تلقائيًا بعد ما تضيف الـ Volume — عادي، انتظر لين يخلص.

بس هذا، صار عندك مساحة تخزين دائمة ما تنمسح أبدًا حتى مع أي تحديث كود مستقبلي. باقي خطوة وحدة بسيطة بالأسفل (متغيّر `GYM_DB_PATH`) عشان نقول للبوت يستخدم هالمساحة.

## الخطوة 3: ضيف المتغيرات (الأسرار)

1. من صفحة المشروع، اضغط على الخدمة (Service) → تبويب **Variables**.
2. اضغط **+ New Variable** وضيف السبعة كلها واحد واحد:

| الاسم | القيمة |
|---|---|
| `TELEGRAM_BOT_TOKEN` | توكن بوتك |
| `TELEGRAM_CHAT_ID` | 449564057 |
| `GOOGLE_CLIENT_ID` | من Google Cloud |
| `GOOGLE_CLIENT_SECRET` | من Google Cloud |
| `GOOGLE_REFRESH_TOKEN` | آخر توكن (اللي فيه الصلاحيات الثلاث) |
| `GEMINI_API_KEY` | مفتاح AI Studio |
| `GYM_DB_PATH` | `/data/gym.db` |

3. بعد الإضافة، Railway يعيد النشر تلقائيًا.

⚠️ لازم تكون سوّيت خطوة الـ Volume (2.5) قبل ما تضيف `GYM_DB_PATH`، وإلا بيانات التمارين ما تنحفظ بشكل دائم.

## الخطوة 4: فعّل الرابط العام

1. بنفس الخدمة → تبويب **Settings** → قسم **Networking**.
2. اضغط **Generate Domain** → بيعطيك رابط مثل:
   `https://fitbit-bot-production-xxxx.up.railway.app`
3. جرب افتح `<الرابط>/ping` بالمتصفح → لازم يطلع **OK**.

## الخطوة 5: حدّث webhook تلقرام

افتح بالمتصفح (بدّل الرابط والتوكن):
```
https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<رابط-Railway>/webhook/<TOKEN>
```
لازم يرد `"Webhook was set"`. من هالحظة البوت يشتغل من Railway! جرب `/start`.

## الخطوة 6: الرسالة الصباحية (خدمة Cron)

1. من صفحة المشروع اضغط **+ Create** → **GitHub Repo** → اختر نفس مستودع `fitbit-bot` (يعني خدمة ثانية من نفس الكود).
2. بالخدمة الجديدة → **Settings**:
   - **Start Command**: `python daily_summary.py`
   - **Cron Schedule**: `0 5 * * *` (يعني 5:00 UTC = 8:00 صباحًا بتوقيت الرياض)
3. تبويب **Variables** بالخدمة الجديدة → ضيف نفس الستة متغيرات (أو استخدم خاصية Shared Variables على مستوى المشروع إن ظهرت لك).

## الخطوة 6.1: تذكير الحركة (خدمة Cron ثانية) 🚶

نفس فكرة الخطوة 6 بالضبط، بس بخدمة جديدة منفصلة:

1. من صفحة المشروع اضغط **+ Create** → **GitHub Repo** → نفس مستودع `fitbit-bot`.
2. بالخدمة الجديدة → **Settings**:
   - **Start Command**: `python inactivity_check.py`
   - **Cron Schedule**: `0 6,9,12,15,18 * * *` (يعني يشتغل 5 مرات باليوم بتوقيت UTC، توافق حوالي 9ص، 12ظ، 3ع، 6م، 9م بتوقيت الرياض)
3. تبويب **Variables** → ضيف نفس المتغيرات (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN`).
4. كل تشغيلة تتحقق: لو خطواتك أقل من المتوقع لهذا الوقت من اليوم، يرسل لك تذكير — وإذا كنت ماشي زين، ما يرسل شي (ما فيه إزعاج بدون داعي).

💡 تقدر تغيّر ساعات الاستيقاظ/النوم أو هدف الخطوات من أول الملف `inactivity_check.py` (المتغيرات `WAKE_HOUR`, `SLEEP_HOUR`, `STEP_GOAL`).

## الخطوة 7: تنظيف القديم

بعد ما تتأكد كل شي يشتغل من Railway:
- من PythonAnywhere → صفحة Web → **Disable** الويب آب (أو خله واحذف الـ scheduled task على الأقل عشان ما توصلك رسالة صباحية مكررة).

# إضافات ذكاء الأداء — جاهزة في هذه النسخة 🧠⚡

هذه النسخة لا تحتاج أي متغيرات سرية جديدة. ارفع الملفات فوق الملفات القديمة في نفس GitHub repository، وRailway سيعيد النشر تلقائيًا.

## الأوامر الجديدة

- `/readiness` — درجة الجاهزية اليومية من 100 حسب النوم، نبض الراحة، النشاط، والحمل التدريبي.
- `/todayplan` — قرار واضح: وش تتمرن اليوم وبأي شدة.
- `/next` — اقتراح الوزن والتكرارات القادمة من سجلك السابق.
- `/progress` — اكتشاف التقدم، الثبات، أو التراجع لكل تمرين.
- `/balance` — تحليل توازن الحمل بين العضلات.
- `/report` — تقرير أسبوعي كامل.

## إضافات تعمل تلقائيًا

- عند فتح أي تمرين في `/gym` يظهر اقتراح وزن الجلسة القادمة واتجاه تقدمك.
- عند تسجيل رقم شخصي جديد يظهر تنبيه PR تلقائيًا ويتحفظ في السجل.
- المدرب الذكي صار يعرف درجة جاهزيتك وحملك التدريبي قبل ما يجاوب.
- `daily_summary.py` يرسل درجة الجاهزية مع الملخص الصباحي، ويضيف التقرير الأسبوعي تلقائيًا صباح الجمعة.
- خدمة `inactivity_check.py` الحالية صارت تنبيهات ذكية شخصية للنوم والنبض والنشاط والجاهزية. **لا تنشئ Cron جديد**؛ احتفظ بنفس خدمة وجدول التنبيه الموجودين عندك.

## مهم جدًا

استمر باستخدام نفس الـ Volume ونفس المتغير:

`GYM_DB_PATH=/data/gym.db`

الجداول الجديدة تنشأ تلقائيًا داخل نفس قاعدة البيانات، ولا تحتاج migration يدوي.
